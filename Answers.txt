LAB 03 – ANSWERS

1. Method Overloading

* Overloading là việc nhiều method có cùng tên nhưng khác tham số (kiểu hoặc số lượng).
* So sánh:

  * Dùng mảng (DigitalVideoDisc[]): phải tạo mảng trước → hơi bất tiện.
  * Dùng varargs (DigitalVideoDisc...): truyền trực tiếp nhiều đối tượng → tiện hơn.
    → Kết luận: varargs tiện và linh hoạt hơn.

2. Passing Parameter

* Java là ngôn ngữ pass by value.
* Giải thích:

  * Khi gọi swap(o1, o2), giá trị tham chiếu được copy → không đổi object gốc.
  * Vì vậy swap không làm thay đổi vị trí 2 object.
* changeTitle:

  * Truyền bản copy của reference → vẫn trỏ đến object thật.
  * Thay đổi thuộc tính → object bị thay đổi.
    → Kết luận: Java luôn truyền tham trị (pass by value).

3. Swap method đúng

* Muốn swap thật sự cần dùng wrapper (array hoặc class chứa object).

4. Debugging

* Breakpoint: điểm dừng chương trình.
* Step Into: vào trong hàm
* Step Over: chạy qua hàm
* Step Return: thoát hàm
* Resume: chạy tiếp

5. Class member vs Instance member

* Class member (static):

  * Dùng chung cho mọi object
  * Ví dụ: nbDigitalVideoDiscs
* Instance member:

  * Riêng từng object
  * Ví dụ: title, cost, id

9. String vs StringBuilder vs StringBuffer

* String:

  * Immutable → tạo nhiều object → chậm
* StringBuilder:

  * Nhanh nhất
  * Không thread-safe
* StringBuffer:

  * Thread-safe
  * Chậm hơn StringBuilder

→ Kết luận:

* Dùng StringBuilder trong hầu hết trường hợp
* Dùng StringBuffer khi cần đa luồng

10. Garbage problem

* Dùng “+” trong loop → tạo nhiều object → tốn bộ nhớ
* Giải pháp:

  * Dùng StringBuilder hoặc StringBuffer
